Project Overview

Built a personal portfolio website and deployed it on GitHub Pages, then configured DNS records across three live domains to point to the site. The project involved understanding how DNS propagation works, the difference between A records and CNAMEs, and how HTTPS certificates are provisioned for custom domains via GitHub Pages.

All three domains are live and resolvable right now. The DNS configuration can be independently verified using public tools like MxToolbox or dig.

Step-by-Step Breakdown
1
Build the Portfolio Site with HTML & CSS
›

Built the site from scratch using HTML, CSS, and minimal JavaScript:

  • Designed a clean, professional layout with sections for experience, skills, projects, and certifications
  • Used semantic HTML for accessibility and good page structure
  • Made it responsive for both desktop and mobile using CSS media queries
  • Kept the site static (no backend required) — suitable for GitHub Pages hosting
2
Deploy to GitHub Pages
›

Deployed the site using GitHub Pages — free static site hosting directly from a GitHub repository:

  • Created a GitHub repository named hi-its-aayush.github.io (special naming convention for user pages)
  • Pushed the site files via Git — git add, git commit, git push
  • Enabled GitHub Pages in the repository settings — set source branch to main
  • Site was immediately accessible at hi-its-aayush.github.io via HTTPS
  • Configured a CNAME file in the repo root with the custom domain — required for GitHub Pages to serve on a custom domain
3
Purchase and Configure the Primary Domain
›

Registered aayushacharya.com.au and configured DNS to point to GitHub Pages:

  • Registered the .com.au domain through a registrar (Namecheap)
  • Accessed the DNS management panel — set TTL to 3600 (1 hour) for initial setup, reduced to 300 during testing
  • Added 4 A records pointing to GitHub Pages' IP addresses (GitHub publishes these officially):
    185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153
  • Added a CNAME record for www subdomain pointing to hi-its-aayush.github.io
  • Set the custom domain in GitHub Pages settings — GitHub automatically provisions an SSL certificate via Let's Encrypt
  • Waited for DNS propagation (~15 minutes to 48 hours depending on TTL) and verified with nslookup aayushacharya.com.au
4
Configure Redirect Domains
›

Set up aayush.com.au and googal.store to redirect to the main domain:

  • Registered both additional domains
  • For aayush.com.au: added a CNAME record at root pointing to aayushacharya.com.au — visitors who type the shorter domain are seamlessly served the same site
  • For googal.store: configured A records pointing to GitHub Pages IPs, added a separate CNAME file in a secondary repo branch
  • Verified both domains resolved correctly and served HTTPS without certificate warnings
  • Tested redirect behaviour — confirmed www.aayush.com.au and aayush.com.au both served the site correctly
5
Verify HTTPS and SSL Certificate
›

Ensured all domains served content over HTTPS with valid certificates:

  • GitHub Pages auto-provisions SSL via Let's Encrypt once the custom domain DNS is correctly configured
  • Enabled "Enforce HTTPS" in GitHub Pages settings — HTTP requests automatically redirect to HTTPS
  • Verified SSL certificate details in browser: issued to the correct domain, valid expiry date, Let's Encrypt CA
  • Checked for mixed content warnings (HTTP resources on an HTTPS page) — none found
  • Used SSL Labs (ssllabs.com) to run an SSL test — confirmed A grade
