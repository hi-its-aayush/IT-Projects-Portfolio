What this proves — 30 second summary

This project demonstrates the ability to build and manage an enterprise Microsoft 365 environment from scratch — the same stack used by mid-to-large companies globally. It covers user identity management, device enrollment, security policy enforcement, and hybrid infrastructure simulation. These are core L1/L2 and junior sysadmin skills that directly translate to day-one productivity in an IT support role.

Project Overview

Using Microsoft's free M365 E5 developer tenant program, I built a production-grade enterprise environment from scratch. The goal was to simulate the full identity, endpoint, and security stack that a corporate IT team manages daily — going beyond just reading documentation by actually deploying and configuring each component.

The project covers three interconnected areas: Identity & Access Management (Entra ID, RBAC, MFA), Endpoint Management (Intune, Autopilot, compliance policies), and Hybrid Infrastructure (virtualised Windows Server 2022 with GPO and network segmentation via Hyper-V).

Step-by-Step Breakdown
1
Set up Microsoft 365 E5 Developer Tenant
›

Enrolled in the Microsoft 365 Developer Program to obtain a free E5 tenant. The E5 license includes the full suite of enterprise tools — Entra ID P2, Intune, Defender, and all M365 apps — making it a realistic simulation of a corporate environment.

  • Created a dedicated tenant domain (e.g. aayushlab.onmicrosoft.com)
  • Assigned global admin credentials and secured with MFA immediately
  • Verified E5 licenses were active across all services in the M365 Admin Centre
2
Configure Entra ID — Users, Groups, and RBAC
›

Built out a realistic user and group structure in Entra ID to simulate a company with multiple departments.

  • Created users across departments (IT, Finance, HR, Management) with consistent naming conventions
  • Built Security Groups and Microsoft 365 Groups for each department
  • Assigned Role-Based Access Control (RBAC) — e.g. helpdesk staff as Helpdesk Administrator, not Global Admin
  • Configured Dynamic Groups using attributes (e.g. department = "Finance") so users auto-join the correct group on creation
  • Set password policies, account lockout thresholds, and self-service password reset (SSPR)
3
Enforce Zero Trust with Conditional Access & Phishing-Resistant MFA
›

Implemented a Zero Trust security posture — no user or device is trusted by default, access is granted only when identity and compliance conditions are met.

  • Created Conditional Access policies: require MFA for all users, block legacy authentication protocols, restrict access to compliant devices only
  • Enforced phishing-resistant MFA (Microsoft Authenticator with number matching, and FIDO2 security key for admin accounts)
  • Configured Named Locations to allow/block access from specific countries or IP ranges
  • Set up Sign-in Risk policies via Entra ID Protection — automatically blocking high-risk sign-ins
  • Tested policies by attempting sign-ins that should be blocked and verifying Entra ID logs showed the correct deny reason
4
Deploy Microsoft Intune MDM — Device Enrollment & Compliance
›

Set up Intune as the Mobile Device Management platform for the simulated fleet.

  • Configured MDM and MAM policies for both corporate and BYOD devices
  • Created Compliance Policies — required BitLocker encryption, minimum OS version, Defender Antivirus enabled, screen lock timeout
  • Built Configuration Profiles to push settings: Wi-Fi profiles, VPN configs, browser settings, and restrictions
  • Assigned profiles to device groups using Entra ID Dynamic Groups
  • Tested Remote Wipe, Remote Lock, and BitLocker key recovery via the Intune portal
5
Configure Windows Autopilot for Zero-Touch Provisioning
›

Configured Autopilot so that a new device shipped directly to a user can be set up without IT ever physically touching it.

  • Created an Autopilot Deployment Profile — User-Driven mode, Azure AD joined, hide OOBE screens
  • Built an Enrollment Status Page (ESP) to ensure apps and policies install before the user hits the desktop
  • Configured required apps to install automatically on enrollment (e.g. Microsoft 365 Apps, Defender, Chrome)
  • Registered a test device hardware hash into Autopilot and validated the full zero-touch flow
  • Set up the Company Portal for optional app self-service
6
Virtualise Windows Server 2022 & Hybrid Infrastructure with Hyper-V
›

Deployed virtualised on-premises infrastructure to simulate a hybrid cloud environment alongside the Entra ID cloud tenant.

  • Installed Hyper-V on a physical machine to host virtual servers
  • Deployed Windows Server 2022 VM and promoted it to a Domain Controller with AD DS role
  • Created an AD domain (aayushlab.local) with OUs mirroring the Entra ID structure
  • Built Group Policy Objects (GPOs): password complexity, mapped drives, desktop lockdown, software restriction policies
  • Configured network segmentation using Hyper-V virtual switches — separated management, domain, and test traffic
  • Set up Entra Connect to sync on-premises AD users to the cloud tenant (hybrid identity)
